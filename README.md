# AI 에이전트 개발 환경 설정

이 레포지토리는 **Claude Code**와 **Gemini**를 활용한 AI 에이전트 개발을 위한 기본 설정과 구현 가이드를 제공합니다. AI 기반 개발에 필요한 기본 구성, 정의, 운영 절차를 정립합니다.

## 1. 목적

이 레포지토리의 주요 목표는 다음과 같습니다:

1. **기본 세팅 제공**: Claude Code와 Gemini를 활용한 AI 에이전트 개발에 필요한 기본 환경 설정을 제공합니다.
2. **서브 에이전트 및 스킬 정의**: Claude Code의 서브 에이전트와 스킬 기능을 기반으로, 개발에 활용할 수 있는 유용한 서브 에이전트와 스킬을 정의합니다.
3. **개발 지침 수립**: Gemini와 Claude Code가 개발 전반에 걸쳐 참고해야 할 지침을 정의합니다.

## 2. 서브 에이전트와 스킬

이 섹션에서는 Claude Code의 기능을 활용한 유용한 서브 에이전트와 스킬을 정의합니다.

### 정의
- **서브 에이전트(Sub-agent)**: 특정 도메인의 작업을 처리하도록 설계된 전문화된 에이전트 프로세스입니다. (예: "브라우저 에이전트", "터미널 에이전트")
- **스킬(Skill)**: 에이전트가 호출할 수 있는 개별 기능 또는 도구 통합입니다. (예: "웹 검색", "파일 조작", "분석")

### 레지스트리
*(이 섹션은 개발이 진행됨에 따라 구체적인 서브 에이전트 구성과 스킬 정의로 채워질 예정입니다.)*

#### 유용한 서브 에이전트
- **리서치 에이전트(Research Agent)**: 외부 소스에서 정보를 수집하고 결과를 요약하는 데 특화되어 있습니다.
- **코딩 에이전트(Coding Agent)**: 구현, 리팩토링, 테스트 생성에 집중합니다.
- **리뷰 에이전트(Review Agent)**: 버그, 보안 문제, 스타일 위반 사항에 대해 코드를 분석합니다.

#### 유용한 스킬
- **컨텍스트 관리(Context Management)**: 대용량 컨텍스트 윈도우를 효율적으로 처리하고 대화 기록을 요약합니다.
- **아티팩트 관리(Artifact Management)**: 구조화된 문서(계획, 작업, 워크스루)를 생성하고 유지합니다.
- **도구 체이닝(Tool Chaining)**: 여러 저수준 도구를 결합하여 고수준 기능을 구현합니다.

### 워크플로우 (Workflows)
정의된 작업 절차를 통해 에이전트 간의 협업을 자동화합니다.

- **Claude-Gemini 협업 워크플로우 (`claude-gemini-cowork.md`)**:
  - **목적**: 기획(Gemini)과 구현(Claude)을 분리하여 품질 높은 코드 작성을 목표로 합니다.
  - **파일 위치**: `.agent/workflows/claude-gemini-cowork.md`
  - **사용법**: 슬래시 커맨드 `/claude-gemini-cowork`를 통해 가이드를 확인하고 프로세스를 시작합니다.

## 3. AI 어시스턴트 지침

이 섹션에서는 이 레포지토리 또는 파생 레포지토리에서 작업할 때 Gemini와 Claude Code가 따라야 할 지침을 설명합니다.

### 핵심 지침
1. **컨텍스트 인식**: 복잡한 작업을 시작하기 전에 항상 현재 디렉토리 구조를 확인하고 관련 문서(`README.md`, `task.md`)를 읽습니다.
2. **사용자 의도 우선**: 사용자의 의도를 우선시합니다. 요청이 모호한 경우 실행 전에 명확한 질문을 합니다.
3. **안전 우선**: 데이터 손실이나 시스템 불안정을 방지하기 위해 모든 파일 작업과 명령 실행을 검증합니다.
4. **에이전트적 행동**: 복잡한 요청에 대해 사용자에게 정보를 제공하고 방향을 맞추기 위해 아티팩트(계획, 작업 목록)를 선제적으로 생성합니다.

### 언어 및 커뮤니케이션
- **기본 언어**: 한국어. 에이전트는 대화, 설명, 문서화에 주로 한국어를 사용합니다.
- **기술 용어**: 정확성을 위해 기술 개념에는 영어 용어를 사용할 수 있지만, 해당되는 경우 표준 한국어 용어를 우선합니다.

### 개발 표준
- **코드 품질**: 깔끔하고, 주석이 달린, 모듈화된 코드를 작성합니다. 언어별 스타일 가이드를 따릅니다. (예: Python의 PEP 8, JavaScript의 ESLint 표준)
- **문서화**: 코드 변경과 동시에 문서(README, 인라인 주석)를 업데이트합니다.
- **아티팩트**: 지속적인 컨텍스트, 계획, 작업 목록을 저장하기 위해 `.gemini/antigravity/brain` 디렉토리를 사용합니다.

### 협업 프로토콜
- 일반 채팅 상호작용과 에이전트 실행 간의 핸드오프가 발생할 때, 상태가 `task.md`에 명확하게 문서화되어 있는지 확인합니다.
- 버전 관리가 활성화된 경우 구체적이고 설명적인 커밋 메시지를 사용합니다.
